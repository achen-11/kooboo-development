# 代码审查检查清单

在交付 Kooboo 代码前，对照以下清单进行自查。

---

## 1. 页面和路由

| 检查项 | 说明 | 状态 |
|--------|------|------|
| 页面有路由定义 | 每个 `.html` 页面文件顶部必须有 `<!-- @k-url /path -->` | [ ] |
| 路由格式正确 | 动态参数使用 `{param}` 格式，如 `/product/{id}` | [ ] |
| 路由不重复 | 检查是否有多个页面定义相同路由 | [ ] |
| 静态路由在前 | 静态路由应放在动态路由前面 | [ ] |

---

## 2. SSR 脚本

| 检查项 | 说明 | 状态 |
|--------|------|------|
| 使用 `env="server"` | 服务端脚本必须添加此属性 | [ ] |
| 数据正确传递 | 使用 `k.state.set()` 或 `k.utils.clientJS.setVariable()` | [ ] |
| 避免敏感操作 | 不要在 SSR 中执行耗时操作 | [ ] |
| 避免客户端敏感数据 | 不要在 SSR 中暴露敏感信息到客户端 | [ ] |

---

## 3. API 定义

| 检查项 | 说明 | 状态 |
|--------|------|------|
| 正确的请求类型 | GET 用 `k.api.get()`，POST 用 `k.api.post()` | [ ] |
| 错误处理 | 返回适当的 HTTP 状态码 (400, 401, 404, 500) | [ ] |
| 参数验证 | 验证必要参数是否存在 | [ ] |
| 输入 sanitization | 对用户输入进行过滤和转义 | [ ] |
| **@k-url 声明** | API 文件顶部必须有 `// @k-url /api/xxx/{action}` | [ ] |
| **Model 引用路径** | 使用 `code/Models`，不是相对路径 | [ ] |
| **同步操作** | k_sqlite 是同步的，不需要 `await` | [ ] |

---

## 4. 数据库操作

### k.DB.sqlite

| 检查项 | 说明 | 状态 |
|--------|------|------|
| SQL 注入防护 | 使用参数化查询，不要拼接字符串 | [ ] |
| 错误处理 | 数据库操作必须 try-catch | [ ] |
| 资源释放 | 确保连接正确关闭 | [ ] |

### k_sqlite ORM

| 检查项 | 说明 | 状态 |
|--------|------|------|
| 模型时间戳 | 使用 `timestamps: true` 启用自动时间戳 | [ ] |
| 软删除 | 使用 `softDelete: true` 启用软删除 | [ ] |
| 索引 | 常用查询字段添加 `index: true` | [ ] |
| 分页 | 大数据量查询使用 `findPaginated` | [ ] |
| 字段选择 | 使用 `select` 指定需要的字段，避免 `SELECT *` | [ ] |
| **不使用 await** | k_sqlite 是同步操作，不需要 `await` | [ ] |
| **使用 _id** | 主键字段是 `_id`，不是 `id` | [ ] |
| **不定义 id** | Model 不需要手动定义 id 字段 | [ ] |

---

## 5. 安全性

| 检查项 | 说明 | 状态 |
|--------|------|------|
| 密码加密 | 存储密码使用 `k.security.sha256()` 或更强算法 | [ ] |
| 敏感数据保护 | 不要在客户端暴露敏感信息 | [ ] |
| 输入验证 | 所有用户输入必须验证 | [ ] |
| SQL 注入 | 使用参数化查询 | [ ] |
| XSS 防护 | 输出时进行 HTML 转义 | [ ] |
| CSRF 防护 | 敏感操作考虑 CSRF token | [ ] |
| 文件上传安全 | 验证文件类型和大小 | [ ] |
| 路径遍历 | 防止路径遍历攻击 | [ ] |

## 自查流程

1. **功能完成**：代码功能开发完成
2. **逐项检查**：对照以上清单逐项检查
3. **问题修复**：发现问题的修复后再提交
4. **最终确认**：所有检查项通过后交付
